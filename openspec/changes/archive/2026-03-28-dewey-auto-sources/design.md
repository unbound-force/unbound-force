## Context

`dewey init` creates a default `sources.yaml` with one
disk source. The developer must manually add sibling
repos. `initSubTools()` already runs `dewey init` and
`dewey index` -- it should generate a rich sources
config between those two steps.

## Goals / Non-Goals

### Goals
- Auto-detect sibling repos in `../` (directories with `.git/`)
- Extract GitHub org from current repo's git remote
- Generate multi-repo `sources.yaml` with disk + GitHub sources
- User-owned after creation (never overwrite customized config)
- Testable with injected dependencies

### Non-Goals
- Detecting repos outside of `../` (e.g., `../../`)
- Validating that detected repos are in the same GitHub org
- Pre-filtering repos without Markdown files (Dewey handles 0-doc sources)
- Managing GitHub authentication (Dewey handles token precedence)

## Decisions

### Detection Logic

```go
func generateDeweySources(opts *Options) {
    sourcesPath := filepath.Join(opts.TargetDir,
        ".dewey", "sources.yaml")

    // Skip if sources.yaml doesn't exist (dewey init didn't run)
    info, err := os.Stat(sourcesPath)
    if err != nil { return }

    // Skip if file was modified by user (not the default).
    // Check: if file has more than the default content
    // (> 1 source entry), assume user customized it.
    data, _ := os.ReadFile(sourcesPath)
    if !isDefaultSourcesConfig(data) { return }

    // Detect sibling repos
    parentDir := filepath.Dir(opts.TargetDir)
    currentName := filepath.Base(opts.TargetDir)
    entries, _ := os.ReadDir(parentDir)

    var siblings []string
    for _, e := range entries {
        if !e.IsDir() { continue }
        if e.Name() == currentName { continue }
        // Check for .git/ directory
        gitDir := filepath.Join(parentDir, e.Name(), ".git")
        if info, err := os.Stat(gitDir); err == nil && info.IsDir() {
            siblings = append(siblings, e.Name())
        }
    }

    // Extract GitHub org from git remote
    org := extractGitHubOrg(opts)

    // Generate sources.yaml
    writeSourcesConfig(sourcesPath, currentName,
        siblings, parentDir, org)
}
```

### isDefaultSourcesConfig

Check if the file content matches the default single-source
pattern. Simple approach: count `- id:` occurrences. If
more than 1, user has customized it.

### Generated sources.yaml Format

```yaml
# Auto-generated by uf init. Customize as needed.
# This file is user-owned -- uf init will not
# overwrite it after initial creation.

sources:
  # Per-repo disk sources (fine-grained provenance)
  - id: disk-local
    type: disk
    name: unbound-force
    config:
      path: "."

  - id: disk-dewey
    type: disk
    name: dewey
    config:
      path: "../dewey"

  # ... one per sibling

  # Org-level files (design papers, plans)
  - id: disk-org
    type: disk
    name: org-workspace
    config:
      path: "../"

  # GitHub API (issues, PRs, READMEs)
  - id: github-org
    type: github
    name: unbound-force org
    config:
      org: unbound-force
      repos:
        - unbound-force
        - dewey
        - gaze
        - website
    refresh_interval: daily
```

### Placement in initSubTools

```
dewey init      ← creates default sources.yaml
generateDeweySources()  ← overwrites with multi-repo config
dewey index     ← indexes all sources
```

## Risks / Trade-offs

**Low risk**: If sibling detection fails (permissions,
no siblings), the default single-source config remains.
If GitHub org extraction fails, the GitHub source is
omitted. The generated config is valid YAML regardless.

**User-owned after creation**: Once the user edits
`sources.yaml`, `uf init` will never overwrite it.
The detection happens only on the first `dewey init`.
