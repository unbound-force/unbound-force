# Data Model: Dewey Unified Memory

**Branch**: `021-dewey-unified-memory`
**Date**: 2026-04-03

## Overview

This spec introduces two new data entities (Learning,
OllamaState) and modifies existing entities (doctor
health checks, setup install commands, agent
instructions). The data model spans three repositories
but is documented here as the authoritative reference.

## Learning Entity (Dewey repo)

A learning is a semantic memory entry stored in Dewey's
`graph.db` alongside specs, code documentation, and web
documentation. It uses the same embedding pipeline and
query infrastructure as all other indexed content.

### Schema

```
Learning {
  // Identity
  id:          string      // Auto-generated UUID (e.g., "lrn-a1b2c3d4")
  content:     string      // The learning text (free-form)

  // Classification
  tags:        []string    // User-provided tags
  source_type: "learning"  // Fixed; distinguishes from "disk", "github", "web"
  source_id:   string      // "learning-{id}" for provenance tracking

  // Temporal
  created_at:  timestamp   // ISO 8601, set at storage time

  // Embedding
  embedding:   []float32   // Generated by granite-embedding:30m (256 dims)
}
```

### Tag Conventions

Tags are free-form strings. The following conventions
are RECOMMENDED for consistency across agents:

| Tag Pattern | Example | Purpose |
|------------|---------|---------|
| Branch name | `021-dewey-unified-memory` | Links learning to feature work |
| Date | `2026-04-03` | Temporal context |
| Category | `pattern`, `gotcha`, `review-insight`, `file-specific` | Classification per Spec 018 R5 |
| File path | `internal/scaffold/scaffold.go` | File-specific learnings |

### Query Patterns

| Query Intent | Tool | Filter |
|-------------|------|--------|
| All learnings | `dewey_semantic_search_filtered` | `source_type: "learning"` |
| Learnings about a file | `dewey_semantic_search` | Query: file path |
| Learnings from a branch | `dewey_semantic_search_filtered` | `has_tag: "branch-name"` |
| Learnings + specs + code | `dewey_semantic_search` | No filter (unified search) |

### MCP Tool: `dewey_store_learning`

**Input**:
```json
{
  "text": "scaffold.go requires initSubTools nil guard for Stdout",
  "tags": ["021-dewey-unified-memory", "2026-04-03", "gotcha", "internal/scaffold/scaffold.go"]
}
```

**Output**:
```json
{
  "id": "lrn-a1b2c3d4",
  "indexed": true
}
```

**Error cases**:
- Embedding model unavailable: Store without embedding
  (keyword-searchable only). Return
  `{ "id": "...", "indexed": true, "embedding": false }`.
- Database write failure: Return error with context.
